Command-line switches

Client

  • -dw<number>: sets screen width to <number>. If height not specified, also sets height to width * 3/4. min is 320
  • -dh<number>: sets screen height to <number>. If width not specified, also sets width to height * 4/3. min is 200
  • -dd<number>: sets z-buffer depth to <number>.
  • -dc<number>: sets colour depth (bits per pixel) to <number>.
  • -da<number>: sets full-scene anti-aliasing (FSAA) to <number> samples.
  • -dv<number>: turns vertical sync on or off. 0 to disable, 1 to enable. Defaults to 0.
  • -ds<number>: sets stencil buffer bits to <number>.
  • -du<number>: set shader quality to <number>. 0 for none, 1 for shaders with fastest quality, max is 3. Defaults to 1 where hardware supports it.
  • -k<number>: enables or disables kid mode. 0 to disable, 1 to enable.
  • -g<gametype>: used to start different games. unused.
  • -h<path>: sets your home directory to <path>. If set, the engine will look for files in your home directory as well as the normal installation directory. However, all files will be written to your home directory, instead of the normal installation directory.
  • -p<path>: includes the data files found in <path>. Used for adding additional mods. Packages will be loaded in the order specified, or if not found, from the installation path.
  • -v<number>: sets verbosity to <number>. This affects how much information is printed to the console. 0 is the least verbose, 4 is the most verbose.
  • -r<filename>: executes the specified init script before OpenGL and various libraries are initialized. If no file specified, executes init.cfg. This setting also causes an init.cfg file to be written out when the game exits to save various settings.
  • -x<filename>: executes script <filename> once the engine has started up and initialized the game.
  • -o<path>: sets the path to your Sauerbraten directory. If Blood Frontier can find your Sauerbraten directory, you can load and play Sauerbraten maps from withing Blood Frontier!

Server

  • -su<number>: sets upstream rate to <number>. Do not set this unless you know what you are doing, it should normally be left out. Use -sc to limit your bandwidth use.
  • -sc<number>: sets the maximum number of clients to <number>.
  • -si<ip>: binds to the ip address <ip>. Only needed for machines with more than one interface.
  • -sm<ip>: uses <ip> as the master server.
  • -sa<number>: contacts master server using port <number>.
  • -sl<something>: sets default map to <something>.
  • -ss<number>: sets server type to <number>. 0 for local only, 1 for private, 2 for public, 3 for dedicated. Defaults to 1.
  • -sp<number>: sets the server port to <number>.
  • -sq<number>: sets the server query port to <number>.
  • -d<description>: sets the server's description to <description>. This will appear in the server list.
  • -k<password>: sets the admin password to <password>.
  • -o<number>: sets the openness of the server to N. N=1 disables mastermodes 2 (locked) and 3 (private), while N=0 (default) enables all mastermodes.

Master server

These are for actually running a masterserver - this is not needed to run a server. Do not use it unless you really know you want your own master server!

  • -ms<number>: something to do with the masterserver, some kind of mode? (check this out)
  • -mi<ip>: binds the masterserver to the interface <ip>. Defaults to localhost.
  • -mp<number>: sets the masterserver to listen on port <number>. Default is 28800.
